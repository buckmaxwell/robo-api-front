Get Feed

request feed:

Endpoint: crimenut.maxwellbuck.com/reports/feed
requires:

  • json with valid token*
  • lon*
  • lat*
  • page
  • miles
returns: list of reports as a json (see ex 5) or error message.

page: this is for pagination. pages are up to 20 reports in length. The first page fetched is “page”:1 (1 is default if page not supplied). If there are next and/or previous pages the response will include a key called next_page and/or prev_page. Suggested use is that when the user is halfway down the page or so the next page is loaded by calling feed on next_page. If the user scrolls for a long time and then scrolls back up, calling feed on prev_page might be intelligent.

miles: this value determines how many miles away from lon and lat that reports will be shown. for example, “miles”:”25” would show reports up to 25 miles away in the feed. Default is 10.

*api required fields. Restrictions on client side could be stricter. Perhaps require time_began and time_ended
